# Project Guidelines

This repository contains the frontend for a university project that simulates a simple banking system.

## Goals

The code must prioritize:

* clarity
* simplicity
* good architecture
* readability

Avoid overly complex solutions.

## Tech Stack

React
TypeScript
Vite
TailwindCSS
Fetch API

Do NOT introduce complex frameworks like Redux or advanced state management libraries.

## Code Style

Prefer:

* small reusable components
* functional components
* clear TypeScript types
* simple service layer for API calls

## Architecture

Follow this separation:

pages → UI pages
components → reusable UI elements
services → API communication
types → TypeScript models

## UI Philosophy

The UI should be:

clean
minimal
easy to understand

Use cards, spacing, and tables.

## Error Handling

Always handle:

* loading states
* API errors
* empty states

## Comments

Explain key logic with comments because this project will be used for teaching purposes.
